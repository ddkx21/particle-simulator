# Алгоритм расчёта сил и конвекции

## 1. Физическая модель

Симулятор моделирует движение водяных капель в масле под действием внешнего электрического поля **E**. Каждая капля приобретает индуцированный дипольный момент, пропорциональный R^3. Капли взаимодействуют через:

1. **Диполь-дипольные силы** (электростатика)
2. **Конвективный перенос** (гидродинамика через тензор Озеена)

---

## 2. Диполь-дипольная сила

### 2.1 Формула точного попарного взаимодействия

Сила на каплю _i_ от капли _j_:

$$

F_i = m_const * (r_i^3 * r_j^3 / |r_ij|^7) * g(r_ij)
$$

где:

- `m_const = 12 * pi * eps_0 * eps_oil * E^2` — константа взаимодействия
- `eps_0 = 8.854e-12 Ф/м` — электрическая постоянная
- `eps_oil = 2.85` — относительная диэлектрическая проницаемость масла
- `E = 3e5 В/м` — напряжённость внешнего поля (направлено по z)
- `r_i, r_j` — радиусы капель
- `r_ij = pos_j - pos_i` — вектор между каплями

### 2.2 Угловая зависимость g(r_ij)

Электрическое поле направлено вдоль оси z. Сила анизотропна:

```
dx, dy, dz = компоненты r_ij
r = |r_ij|

factor = m_const * r_i^3 * r_j^3 / r^7

ang_xy = 4*dz^2 - dx^2 - dy^2

F_x = factor * dx * ang_xy
F_y = factor * dy * ang_xy
F_z = factor * dz * (2*dz^2 - 3*dx^2 - 3*dy^2)
```

Физический смысл: капли, выстроенные вдоль поля (z), притягиваются; расположенные перпендикулярно — отталкиваются.

### 2.3 Прямой метод (O(N^2))

Файл: `force_calculator/direct_droplet_force_calculator.py`

Для каждой пары (i, j) с i != j:

1. Вычислить r_ij с учётом периодики (если включена)
2. Если |r_ij| > r_i + r_j — вычислить силу по формуле выше
3. Иначе — зафиксировать столкновение

### 2.4 Метод октодерева Барнса-Хатта (O(N log N))

Файл: `octree/flat_tree.py`

#### Построение дерева (level-by-level)

Три фазы на каждом уровне:

1. **assign_octants** — параллельно: каждая частица определяет свой октант в текущем узле
2. **subdivide_and_alloc** — последовательно: узлы с count > mpl разбиваются на 8 дочерних
3. **scatter** — параллельно: частицы перемещаются в дочерние узлы

Процесс повторяется до тех пор, пока все листья содержат <= mpl частиц.

#### Агрегация R^3 (bottom-up)

Для каждого узла хранятся:

- `R3_sum = sum(r_j^3)` — суммарный "объём" поддерева
- `R3_cx, R3_cy, R3_cz = sum(pos_j * r_j^3)` — взвешенные координаты

Центр масс: `center = (R3_cx, R3_cy, R3_cz) / R3_sum`

#### Обход дерева (stackless walk)

Для каждой частицы i параллельно:

1. Начать с корня (node_idx = 0)
2. **Лист**: точное попарное взаимодействие со всеми частицами листа
3. **Внутренний узел**: проверить критерий открытия Барнса-Хатта:
   - `s = max(width, height, depth)` — размер узла
   - `D = |pos_i - center|` — расстояние до центра масс
   - Если `s^2 < theta^2 * D^2` — **аппроксимировать**: подставить R3_sum и center в формулу силы
   - Иначе — **раскрыть**: перейти к дочерним узлам

Аппроксимация заменяет группу частиц одной "эффективной каплей" с радиусом R3_eff^(1/3) в центре масс.

## 3. Конвекция (тензор Озеена / Стоксовский поток)

### 3.1 Физика

Сила на каплю j создаёт течение в вязкой среде. Это течение переносит другие капли. Скорость в точке r от точечной силы F в начале координат:

$$
v(r) = G(r) * F
$$

где G — тензор Озеена (Oseen tensor):

$$
G_ab(r) = (1 / (8*pi*eta_oil)) * (delta_ab / |r| + r_a * r_b / |r|^3)
$$

### 3.2 Прямой расчёт (O(N^2))

Файл: `force_calculator/direct_droplet_force_calculator.py`, строки 229-254

Для каждой пары (i, j), i != j:

```
r = pos_i - pos_j     (вектор от источника к наблюдателю)
|r| = sqrt(r . r)
eta_const = 1 / (8 * pi * eta_oil)

V_x = eta_const * ((1/|r| + x^2/|r|^3)*Fx + xy/|r|^3 * Fy + xz/|r|^3 * Fz)
V_y = eta_const * (xy/|r|^3 * Fx + (1/|r| + y^2/|r|^3)*Fy + yz/|r|^3 * Fz)
V_z = eta_const * (xz/|r|^3 * Fx + yz/|r|^3 * Fy + (1/|r| + z^2/|r|^3)*Fz)
```

Компактная форма:

$$
v = eta_const / |r| * (F + (F . r_hat) * r_hat / |r|^2)
$$

### 3.3 Октодерево: монополь + дипольная коррекция

Файл: `octree/flat_tree.py`, строки 669-828

#### Монопольный вклад (для внутренних узлов)

Та же формула, что и прямой метод, но F = суммарная сила всего поддерева, r = вектор до центра масс:

$$
F_eff = (force_sum_x, force_sum_y, force_sum_z)
$$

$$
v_monopole = eta_const / |r| * (F_eff + (F_eff . r_hat) * r_hat)
$$

#### Дипольная коррекция (второй порядок мультиполя)

Учитывает, что частицы в узле _распределены_, а не сосредоточены в одной точке.

**Шаг 1: Тензор силовых моментов** (вычисляется при bottom-up агрегации):

```
P_ab = sum_j (pos_j_a * F_j_b)    (9 компонент, a,b = x,y,z)
```

**Шаг 2: Дипольный тензор** (вычисляется on-the-fly при обходе):

```
D_ab = P_ab - cm_a * F_eff_b
```

где cm = центр масс (R^3-взвешенный)

**Шаг 3: Градиент тензора Озеена**

```
u = D^T * r          (вектор)
w = D * r            (вектор)
s_tr = trace(D)      (скаляр)
q = r^T * D * r      (скаляр)

delta_v_a = eta_const * (
    (u_a - w_a - s_tr * r_a) / |r|^3
    + 3 * r_a * q / |r|^5
)
```

Итого: `v = v_monopole + delta_v`

## 4. Интегрирование по времени

Файл: `solver/euler_droplet_solver.py`

Метод Эйлера (первый порядок):

```
1. forces = calculate(positions, radii)                    # диполь-дипольные силы
2. v_migration = forces / (stokes_factor * radii)          # миграционная скорость
   stokes_factor = 2*pi*eta_oil*(2*eta_oil+3*eta_water)/(eta_oil+eta_water)
3. v_convection = calculate_convection(positions, radii, forces)  # конвективная скорость
4. positions += (v_migration + v_convection) * dt           # шаг Эйлера
5. Применить граничные условия (periodic: positions %= L)
```

При столкновении (|r_ij| < r_i + r_j) капли объединяются с сохранением объёма.

## 5. Параметры точности октодерева

| Параметр | Описание                       | Влияние                                                               |
| -------- | ------------------------------ | --------------------------------------------------------------------- |
| `theta`  | Критерий открытия Барнса-Хатта | Меньше = точнее, но медленнее. 0.15 — высокая точность, 0.5 — быстро  |
| `mpl`    | Макс. частиц в листе           | Больше = меньше глубина дерева, но больше точных вычислений в листьях |
